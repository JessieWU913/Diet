# 演示流程页（导师答辩版）

## 1. 演示目标

在 5~8 分钟内展示系统核心价值：可用、可解释、可扩展。

## 2. 演示前准备

- 后端服务已启动，Neo4j 已连接
- 前端页面可访问
- 准备一个普通账号与一个管理员账号

## 3. 主流程（建议按顺序）

### 步骤 1：登录与用户画像

- 动作：登录后进入个人信息页，填写身高/体重/目标体重/减脂周期、忌口与偏好
- 讲解点：系统会将画像用于后续推荐与约束

### 步骤 2：AI 对话推荐

- 动作：在聊天页提问减脂饮食建议
- 讲解点：系统会结合长期记忆、近期饮食、当前模式生成建议

### 步骤 3：导出菜谱到食谱页

- 动作：点击导出并跳转食谱页查看推荐
- 讲解点：支持筛选、详情查看与结果复用

### 步骤 4：记录饮食与运动

- 动作：在饮食记录页添加餐次，在健康管理页添加运动消耗
- 讲解点：摄入与消耗会共同影响当天剩余预算

### 步骤 5：查看健康管理与日志

- 动作：查看健康管理面板与健康日志时间线
- 讲解点：展示数据闭环与趋势追踪能力

### 步骤 6：管理员视角（可选）

- 动作：登录管理页，查看总览/导入数据
- 讲解点：系统具备后台运维与数据治理能力

## 4. 每步推荐话术（可直接念）

- “这里先设定用户目标，后续推荐都会受这些约束影响。”
- “当前回答不是固定模板，而是结合图谱和记忆动态生成。”
- “导出的菜谱会进入可操作页面，便于后续记录与复盘。”
- “这里展示了摄入、消耗和预算的联动关系，形成闭环。”

## 5. 常见提问与回答要点

- 问：如何保证推荐可靠？
  答：优先走图谱事实查询，减少纯文本臆测。
- 问：如何支持个性化？
  答：通过用户画像 + 反馈记忆 + 情景记忆进行约束。
- 问：如何扩展？
  答：新增接口由 views + urls 接入，新增图谱字段由导入脚本与查询层扩展。

## 6. 演示失败兜底方案

- 若模型接口波动：演示历史会话与已导出样例
- 若 Neo4j 不可用：演示前端流程与 API 规范文档
- 若网络不稳定：切换本地录屏并讲解架构图
